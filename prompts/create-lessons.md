## 단원 1: Prompt Engineering 개요 (intro)

### 학습 목표
- Prompt Engineering의 정의와 정책 연구에서의 중요성 이해
- 좋은 프롬프트의 3가지 원칙 학습
- AI를 정책 분석 도구로 활용하는 관점 형성

### 포함 내용
1. **Prompt Engineering이란?**
   - AI에게 효과적인 지시를 내리는 기술
   - 정책 연구자를 위한 새로운 연구 도구

2. **왜 정책학도에게 중요한가?**
   - 문헌 검토 효율화
   - 정책 브리핑 및 보고서 초안 작성
   - 데이터 해석 및 인사이트 도출
   - 다국어 정책 자료 분석

3. **나쁜 프롬프트 vs 좋은 프롬프트 비교**
   - 나쁜 예: "저출산 정책 분석해줘"
   - 좋은 예: "한국의 저출산 대응 정책 중 '육아휴직 급여 인상'에 대해 Bardach의 8단계 정책분석 프레임워크를 적용하여 분석해주세요. 정책 목표, 대안, 예상 효과를 포함해주세요."

4. **좋은 프롬프트의 3원칙**
   - **명확성 (Clarity)**: 분석 대상, 범위, 관점 구체화
   - **맥락 (Context)**: 이론적 프레임워크, 배경 정보 제공
   - **형식 (Format)**: 원하는 출력 구조 명시 (표, 개조식, 에세이 등)

5. **이 과정에서 배울 내용**
   - Zero-shot: 빠른 정책 요약, 개념 설명
   - Few-shot: 일관된 형식의 정책 사례 분석
   - Chain of Thought: 복잡한 정책 효과 추론

### 예제 프롬프트
1. **정책 브리핑 작성**
   - 특정 정책에 대한 1페이지 브리핑 작성 요청
   - 배경, 현황, 쟁점, 권고사항 구조

2. **문헌 검토 구조화**
   - 연구 주제에 대한 문헌 검토 틀 요청
   - 이론적 관점, 실증 연구, 연구 갭 식별

## 단원 2: Zero-shot Prompting

### 학습 목표
- Zero-shot의 개념 이해
- 정책 연구에서의 활용 상황 파악
- 효과적인 Zero-shot 프롬프트 작성법

### 포함 내용
1. **Zero-shot이란?**
   - 예시 없이 지시만으로 작업 수행
   - AI가 이미 학습한 지식을 활용

2. **정책 연구에서 언제 사용하나?**
   - 정책 문서 요약 및 번역
   - 개념 정의 및 설명 요청
   - 이해관계자 목록 작성
   - 정책 용어 해설
   - SWOT, PEST 등 기본 프레임워크 적용

3. **정책 분야 활용 예시**
   ```
   다음 OECD 보고서의 핵심 내용을 3가지 bullet point로 요약해주세요.
   대상 독자는 한국 기획재정부 공무원입니다.
   
   [보고서 내용]
   ```

4. **효과적인 작성법**
   - 분석 대상 명확히 지정
   - 출력 형식 구체화 (표, bullet point, 에세이)
   - 대상 독자 명시
   - 분량 제한 설정

5. **한계점**
   - 복잡한 다단계 분석에는 부적합
   - 일관된 형식의 반복 작업에는 Few-shot이 효과적
   - 최신 정책 정보는 반영되지 않을 수 있음

### 예제 프롬프트

1. **정책 이해관계자 분석**
   ```
   한국의 '국민연금 개혁'에 대한 주요 이해관계자를 분석해주세요.
   
   다음 형식으로 작성:
   | 이해관계자 | 입장 | 주요 관심사 | 영향력 |
   
   최소 5개 이해관계자 그룹을 포함해주세요.
   ```

2. **개발지표 해석**
   ```
   다음 국가의 HDI(인간개발지수) 데이터를 해석하고, 
   정책적 시사점을 도출해주세요.
   
   - 국가: 베트남
   - HDI: 0.703 (2021) → 0.726 (2023)
   - 구성요소: 기대수명 75.4세, 평균교육연수 8.2년, 1인당 GNI $8,660
   
   UNDP의 HDI 분류 기준을 참고하여 분석해주세요.
   ```

### 실습
**과제**: 다음 프롬프트를 개선하여 더 유용한 결과를 얻어보세요.

시작 프롬프트:
```
기후변화 정책 설명해줘
```

**힌트**:
- 어떤 국가 또는 지역의 정책인가요?
- 어떤 측면(완화/적응/재정)에 초점을 맞출까요?
- 누구를 위한 설명인가요? (학부생, 정책입안자, 일반시민)
- 어떤 형식으로 받고 싶나요?

## 단원 3: Few-shot Prompting

### 학습 목표
- Few-shot의 개념과 Zero-shot과의 차이 이해
- 정책 분석에서 일관된 형식 유지하기
- 사례 비교 분석에 Few-shot 활용하기

### 포함 내용
1. **Few-shot이란?**
   - 몇 가지 예시를 통해 AI에게 패턴 학습 유도
   - "이런 형식으로 해줘"를 예시로 보여주는 방식

2. **Zero-shot vs Few-shot 비교**
   | 구분 | Zero-shot | Few-shot |
   |------|-----------|----------|
   | 예시 제공 | 없음 | 2-5개 |
   | 적합한 상황 | 단순 작업, 범용적 형식 | 특정 형식, 일관성 필요 |
   | 정책 활용 예 | 개념 설명, 요약 | 사례 분류, 비교 분석 |

3. **정책 연구에서 언제 사용하나?**
   - 여러 국가/사례의 정책을 동일한 기준으로 비교
   - 정책 문서를 일관된 형식으로 요약
   - 정책 성공/실패 사례 분류
   - 연구 논문 초록 구조화

4. **Few-shot 프롬프트 구조**
   ```
   [작업 설명]
   
   예시 1:
   입력: [정책 A]
   출력: [분석 결과 A]
   
   예시 2:
   입력: [정책 B]
   출력: [분석 결과 B]
   
   실제 작업:
   입력: [분석할 정책]
   출력:
   ```

5. **좋은 예시 선택 팁**
   - **다양성**: 서로 다른 유형의 정책 포함
   - **대표성**: 전형적인 사례 선택
   - **일관성**: 동일한 분석 구조 유지
   - **적절한 수**: 2-5개 (너무 많으면 혼란)

6. **주의사항**
   - 예시의 품질이 결과 품질을 결정
   - 편향된 예시는 편향된 분석을 유발
   - 예시 작성에 시간 투자 필요

### 예제 프롬프트

1. **정책 사례 분류 (성공/실패 요인 분석)**
   ```
   다음 정책 사례들을 분석하고, 새로운 사례도 같은 형식으로 분석해주세요.
   
   예시 1:
   정책: 르완다 의료보험 개혁 (Mutuelles de Santé)
   결과: 성공
   핵심 요인: 강력한 정치적 의지, 커뮤니티 기반 접근, 단계적 확대
   교훈: 지역사회 참여가 보편적 의료보장의 핵심
   
   예시 2:
   정책: 베네수엘라 가격통제 정책 (2003-2013)
   결과: 실패
   핵심 요인: 시장 왜곡, 공급 부족 초래, 암시장 형성
   교훈: 가격통제는 공급측 인센티브를 고려해야 함
   
   분석할 정책:
   정책: 인도네시아 연료보조금 개혁 (2015)
   결과:
   핵심 요인:
   교훈:
   ```

2. **정책 브리핑 표준화**
   ```
   다음 형식에 맞춰 정책 브리핑을 작성해주세요.
   
   예시:
   정책명: 한국 그린뉴딜 (2020)
   ---
   [배경] 코로나19 경제위기 + 기후변화 대응 필요성
   [목표] 2025년까지 73.4조원 투자, 일자리 65.9만개 창출
   [핵심 과제] ①그린 리모델링 ②그린 에너지 ③그린 모빌리티
   [쟁점] 재원 조달 방안, 기존 산업 전환 지원
   [평가] OECD 그린회복 평가에서 긍정적 평가
   ---
   
   작성할 정책: EU 탄소국경조정메커니즘 (CBAM)
   ```

### 실습
**과제**: Few-shot 프롬프트를 작성하여 SDGs(지속가능발전목표) 관련 국가별 정책을 비교 분석해보세요.

- 분석 항목을 더 구체화할 수 있나요? (예: 예산, 도전과제)
- 개발도상국 사례도 포함해보세요

## 단원 4: Chain of Thought (CoT)

### 학습 목표
- CoT의 개념과 복잡한 정책 분석에서의 효과 이해
- Zero-shot CoT vs Few-shot CoT 구분
- 정책 효과 추론, 비용-편익 분석에 CoT 적용

### 포함 내용
1. **Chain of Thought란?**
   - AI에게 단계별 사고 과정을 명시적으로 요청
   - "사고의 연쇄" - 중간 추론 과정을 보여주도록 유도
   - 복잡한 정책 문제를 작은 단계로 분해

2. **왜 정책 분석에 효과적인가?**
   - 정책 효과의 인과관계 추적
   - 다중 이해관계자의 반응 예측
   - 의도하지 않은 결과(unintended consequences) 식별
   - 논리적 오류 감소 및 검증 가능

3. **기본 사용법**
   
   **방법 1: 직접 요청**
   ```
   이 정책의 효과를 단계별로 분석해주세요.
   ```
   
   **방법 2: "Let's think step by step" 추가**
   ```
   최저임금 인상이 고용에 미치는 영향을 분석해주세요.
   Let's think step by step.
   ```
   
   **방법 3: 분석 단계 명시**
   ```
   다음 단계에 따라 분석해주세요:
   1단계: 정책의 직접적 효과 식별
   2단계: 이해관계자별 반응 예측
   3단계: 2차 효과 및 파급효과 분석
   4단계: 장기적 균형 상태 추론
   5단계: 종합 평가 및 정책 제언
   ```

4. **Zero-shot CoT vs Few-shot CoT**
   | 구분 | Zero-shot CoT | Few-shot CoT |
   |------|---------------|--------------|
   | 방법 | "step by step" 키워드 추가 | 추론 과정 예시 제공 |
   | 장점 | 매우 간단 | 더 정교한 분석 |
   | 적합한 상황 | 일반적인 정책 분석 | 특정 분석 프레임워크 적용 |

5. **정책 분야 적용**
   - **정책 효과 추론**: 인과관계 사슬 분석
   - **비용-편익 분석**: 단계별 비용/편익 항목 도출
   - **이해관계자 분석**: 각 그룹의 반응 순차적 예측
   - **시나리오 분석**: 조건별 결과 추론
   - **정책 대안 평가**: 기준별 체계적 비교

### 예제 프롬프트

1. **정책 효과 인과관계 분석 (Zero-shot CoT)**
   ```
   한국 정부가 '주 52시간 근무제'를 도입했을 때 예상되는 효과를 
   단계별로 분석해주세요.
   
   다음 관점을 포함해주세요:
   - 근로자 측면
   - 기업 측면  
   - 거시경제 측면
   - 사회문화적 측면
   
   각 단계에서의 인과관계를 명확히 설명하고,
   의도하지 않은 결과(unintended consequences)도 고려해주세요.
   
   Let's think step by step.
   ```

2. **정책 대안 비교 분석 (Few-shot CoT)**
   ```
   다음 예시처럼 정책 대안을 단계별로 비교 분석해주세요.
   
   예시:
   문제: 대도시 교통 혼잡
   대안 1: 혼잡통행료 도입
   
   분석:
   1단계 (직접 효과): 통행료 부과 → 승용차 이용 비용 증가
   2단계 (행동 변화): 일부 운전자 대중교통 전환, 출퇴근 시간 조정
   3단계 (시스템 효과): 혼잡 감소, 통행 속도 향상, 대중교통 수요 증가
   4단계 (2차 효과): 대중교통 혼잡 증가 가능, 형평성 이슈 제기
   5단계 (장기 효과): 도시 구조 변화, 직주근접 촉진 가능
   
   종합 평가: 효과성 높으나 형평성 보완 필요
   
   ---
   분석할 정책:
   문제: 개발도상국 농촌 빈곤
   대안: 조건부 현금이전 프로그램 (CCT)
   
   분석:
   1단계 (직접 효과):
   2단계 (행동 변화):
   3단계 (시스템 효과):
   4단계 (2차 효과):
   5단계 (장기 효과):
   
   종합 평가:
   ```
