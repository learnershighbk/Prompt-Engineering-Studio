## 1. 프로젝트 개요

### 1.1 배경
Prompt Engineering은 AI 활용의 핵심 역량으로 부상하고 있으나, 개념을 학습하고 직접 실습할 수 있는 통합 환경이 부족하다. 특히 공공정책, 개발정책, 행정학 분야의 대학원생들이 연구와 정책 분석에 AI를 효과적으로 활용하기 위한 맞춤형 학습 환경이 필요하다. Scrimba와 같은 인터랙티브 학습 방식을 참고하여, 정책학 맥락의 이론 학습과 실습을 한 곳에서 수행할 수 있는 웹서비스를 구축한다.

### 1.2 목적
- Prompt Engineering의 기본 개념과 주요 기법을 **정책 분석 및 연구 맥락**에서 체계적으로 학습
- 정책 브리핑, 문헌 검토, 이해관계자 분석 등 **정책학 과제에 바로 활용 가능한** 실습 환경 제공
- 대학원 학생 및 직원들이 학번/사번만으로 쉽게 접근할 수 있는 간소화된 서비스 제공

### 1.3 목표 사용자

#### 주요 대상: 정책대학원 석사과정 학생
| 프로그램 | 영문명 | 주요 관심 분야 |
|----------|--------|---------------|
| 공공정책학 석사 | Master of Public Policy (MPP) | 정책 분석, 정책 평가, 공공재정 |
| 개발정책학 석사 | Master of Development Policy (MDP) | 국제개발협력, SDGs, ODA |
| 공공관리학 석사 | Master of Public Management (MPM) | 공공조직관리, 거버넌스, 행정개혁 |
| 데이터사이언스·정책관리 석사 | Master of Data Science and Public Policy & Management | 데이터 기반 정책 분석, 증거기반정책 |

#### 사용자 구분
| 구분 | 설명 |
|------|------|
| 대학원 학생 (한국인) | 학번으로 식별, 정책 분석 및 연구에 AI 활용 |
| 대학원 학생 (외국인) | 학번으로 식별, 영어 인터페이스 제공, 국제개발 이슈 관심 |
| 교직원 | 사번으로 식별, 정책 보고서 작성 및 업무 효율화에 AI 활용 |

#### 학습자 특성
- 정책 분석, 정책 평가, 공공관리에 대한 기본 지식 보유
- 연구 논문, 정책 브리핑, 사례 분석 등의 과제 수행
- 개발도상국 이슈, 국제개발협력에 관심 (특히 MDP)
- 데이터 기반 정책 분석 및 의사결정에 대한 이해
- 영어와 한국어 모두 사용하는 국제적 환경

## 5. 제약사항

### 5.1 기술적 제약
- 프론트엔드: Next.js 14+ (App Router)
- 백엔드: Next.js API Routes
- 데이터베이스: Supabase (PostgreSQL)
- AI API: OpenAI API (GPT-4o-mini 또는 GPT-4o)
- 배포: Vercel

### 5.2 운영 제약
- MVP 단계에서는 복잡한 회원가입/인증 절차 배제
- 학번/사번은 9자리 숫자로 고정
- AI API 비용 관리를 위한 사용량 제한 고려 필요

## 6. 학습 콘텐츠 범위 (MVP)

### 6.1 필수 학습 단원
| 순서 | 단원명 | 설명 | 학습 시간(예상) |
|------|--------|------|----------------|
| 1 | Prompt Engineering 개요 | 정의, 중요성, 기본 원칙 (정책 분석 관점) | 10분 |
| 2 | Zero-shot Prompting | 예시 없이 지시하는 방법 (정책 요약, 개념 설명) | 15분 + 실습 |
| 3 | Few-shot Prompting | 예시를 통한 패턴 학습 (정책 사례 비교 분석) | 15분 + 실습 |
| 4 | Chain of Thought (CoT) | 단계별 추론 유도 (정책 효과 인과관계 분석) | 15분 + 실습 |

**총 학습 시간: 약 1시간 이내**

### 6.2 콘텐츠 특성
- **정책학 맥락**: 모든 예제와 실습이 정책 분석, 정책 평가, 개발협력 관련
- **학술적 활용**: 문헌 검토, 연구 논문, 정책 브리핑 등 과제에 바로 활용 가능
- **실제 사례**: SDGs, ODA, 거버넌스, 사회정책 등 실제 정책 이슈 다룸
- **다국어**: 한국어/영어 콘텐츠 모두 정책학 용어 및 맥락 반영

### 6.3 향후 확장 고려 단원
- Role Prompting (정책 전문가 역할 부여)
- Self-Consistency (정책 대안 검증)
- Tree of Thought (복잡한 정책 문제 분석)
- ReAct Prompting (정책 정보 검색 + 추론)
- Prompt Chaining (다단계 정책 분석)

## 7. 용어 정의

| 용어 | 정의 |
|------|------|
| Prompt | AI 모델에 전달하는 입력 텍스트 |
| Prompt Engineering | 효과적인 AI 응답을 얻기 위해 프롬프트를 설계하는 기술 |
| Zero-shot | 예시 없이 작업을 지시하는 방식 |
| Few-shot | 소수의 예시를 포함하여 작업을 지시하는 방식 |
| Chain of Thought | AI가 단계별로 추론하도록 유도하는 방식 |
| Playground | 자유롭게 프롬프트를 실험할 수 있는 공간 |
