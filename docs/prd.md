# Product Requirements Document (PRD)

## 프로젝트명
Prompt Lab - Prompt Engineering 학습 플랫폼

---

## 1. 제품 개요

### 1.1 제품 비전
**공공정책, 개발정책, 공공관리학 석사과정 학생**이 복잡한 가입 절차 없이 학번/사번만으로 접속하여, Prompt Engineering의 핵심 기법을 **정책 분석 맥락**에서 1시간 내에 학습하고 직접 실습할 수 있는 인터랙티브 웹서비스

### 1.2 핵심 가치
| 가치 | 설명 |
|------|------|
| 즉시 접근 | 학번/사번 입력만으로 바로 시작 |
| 핵심 집중 | 3가지 핵심 기법(Zero-shot, Few-shot, CoT)에 집중 |
| 실습 중심 | 이론 학습 후 즉시 AI와 실습 |
| 정책 맥락 | 정책 분석, 문헌 검토, 사례 비교 등 정책학 과제에 바로 활용 |

### 1.3 대상 사용자
| 프로그램 | 주요 활용 분야 |
|----------|---------------|
| MPP (공공정책학) | 정책 분석, 정책 평가, 공공재정 분석 |
| MDP (개발정책학) | ODA 효과성, SDGs 분석, 개발 사례 연구 |
| MPM (공공관리학) | 조직 분석, 거버넌스, 행정개혁 사례 |
| DS&PM (데이터사이언스·정책관리) | 데이터 해석, 증거기반정책, 정량분석 |

### 1.4 성공 기준
| 지표 | 목표 |
|------|------|
| 첫 화면 → 학습 시작 | 30초 이내 |
| 학습 완료율 | 70% 이상 |
| 사용자당 프롬프트 실행 | 평균 10회 이상 |
| 과제 활용률 | 학습 후 정책 과제에 활용 시도 |

## 2. 사용자 플로우

### 2.1 전체 플로우
```
[첫 접속]
    ↓
[학번/사번 입력] ──→ [신규: 자동 계정 생성]
    ↓                     ↓
[기존 사용자]         [users 테이블 저장]
    ↓                     ↓
    └────────┬────────────┘
             ↓
      [대시보드/학습 목록]
             ↓
    ┌────────┴────────┐
    ↓                 ↓
[학습 모드]      [Playground]
    ↓                 ↓
[개념 학습]      [자유 실습]
    ↓
[단원 내 실습]
    ↓
[진도 저장]
```

### 2.2 화면별 상세

#### 2.2.1 로그인 화면 (`/`)
| 요소 | 상세 |
|------|------|
| 타이틀 | "Prompt Lab" |
| 서브타이틀 | "학번 또는 사번을 입력하세요" / "Enter your student or employee ID" |
| 입력 필드 | 숫자 입력, 9자리 고정 |
| 언어 선택 | 한글(KOR) / English(ENG) 토글 |
| 버튼 | "시작하기" / "Get Started" |
| 동작 | 입력값 검증 → DB 조회/생성 → `/learn`으로 이동 |

#### 2.2.2 학습 목록 화면 (`/learn`)
| 요소 | 상세 |
|------|------|
| 헤더 | 사용자 ID 표시, Playground 바로가기 |
| 학습 카드 목록 | 4개 단원 카드 형태로 표시 |
| 카드 정보 | 단원명, 설명, 예상 시간, 완료 여부 뱃지 |
| 진도 표시 | 상단에 "2/4 완료" 형태 |

#### 2.2.3 학습 상세 화면 (`/learn/[slug]`)
| 요소 | 상세 |
|------|------|
| 레이아웃 | 좌측: 학습 콘텐츠 / 우측: 실습 영역 (데스크톱) |
| 모바일 | 상단: 콘텐츠 / 하단: 실습 (탭 전환) |
| 콘텐츠 영역 | 마크다운 렌더링, 예제 코드 블록 |
| 실습 영역 | 프롬프트 입력창, 실행 버튼, 응답 표시 |
| 하단 | 이전/다음 단원 버튼, "완료" 체크박스 |

#### 2.2.4 Playground 화면 (`/playground`)
| 요소 | 상세 |
|------|------|
| 레이아웃 | 좌측: 입력 / 우측: 출력 (데스크톱) |
| 입력 영역 | textarea (최대 4,000자), 글자 수 표시 |
| 실행 버튼 | "프롬프트 실행" |
| 출력 영역 | AI 응답 실시간 스트리밍 표시 |
| 부가 기능 | 초기화 버튼, 복사 버튼 |

## 3. 기능 명세

### 3.1 인증 기능

#### F-AUTH-01: 학번/사번 입력 및 검증
| 항목 | 내용 |
|------|------|
| 입력 | 9자리 숫자 (고정) |
| 검증 | 숫자만 허용, 정확히 9자리 |
| 에러 메시지 | "학번 또는 사번은 9자리 숫자입니다" / "ID must be exactly 9 digits" |

#### F-AUTH-02: 자동 계정 생성/조회
| 항목 | 내용 |
|------|------|
| 조회 | `users` 테이블에서 `student_id` 검색 |
| 신규 | 없으면 자동 INSERT |
| 응답 | `user_id` (UUID) 반환 |
| 세션 | 쿠키 또는 localStorage에 저장 |

#### F-AUTH-03: 언어 설정
| 항목 | 내용 |
|------|------|
| 지원 언어 | 한글(KOR), English(ENG) |
| 기본값 | 한글(KOR) |
| 저장 | `users` 테이블 `language` 컬럼 + localStorage |
| 적용 범위 | 전체 UI 텍스트, 학습 콘텐츠 |
| 전환 | 헤더에서 언제든 변경 가능 |

### 3.2 학습 기능

#### F-LEARN-01: 학습 목록 조회
| 항목 | 내용 |
|------|------|
| 데이터 소스 | `data/lessons.json` (정적) |
| 진도 조회 | `progress` 테이블에서 사용자별 완료 상태 |
| 응답 | 단원 목록 + 완료 여부 |

#### F-LEARN-02: 학습 콘텐츠 표시
| 항목 | 내용 |
|------|------|
| 콘텐츠 형식 | Markdown |
| 렌더링 | `react-markdown` 또는 `next-mdx-remote` |
| 예제 표시 | 코드 블록 + "복사" 버튼 |

#### F-LEARN-03: 학습 완료 처리
| 항목 | 내용 |
|------|------|
| 트리거 | "완료" 체크박스 클릭 |
| 동작 | `progress` 테이블 UPSERT |
| 피드백 | 완료 뱃지 표시, 진도 업데이트 |

### 3.3 실습 기능

#### F-PLAY-01: 프롬프트 실행
| 항목 | 내용 |
|------|------|
| 입력 | 프롬프트 텍스트 (최대 4,000자) |
| API | OpenAI API (GPT-4o-mini 또는 GPT-4o) |
| 응답 | 스트리밍 방식으로 실시간 표시 |
| 에러 처리 | API 오류 시 사용자 친화적 메시지 |

#### F-PLAY-02: 응답 표시
| 항목 | 내용 |
|------|------|
| 표시 방식 | 마크다운 렌더링 |
| 스트리밍 | 글자 단위 실시간 출력 |
| 복사 | "복사하기" 버튼 |

## 4. 학습 콘텐츠 구조

### 4.1 단원 목록
| slug | 단원명 | 순서 | 정책학 활용 |
|------|--------|------|------------|
| `intro` | Prompt Engineering 개요 | 1 | 정책 분석 도구로서의 AI 이해 |
| `zero-shot` | Zero-shot Prompting | 2 | 정책 요약, 개념 설명, 이해관계자 분석 |
| `few-shot` | Few-shot Prompting | 3 | 정책 사례 비교, 일관된 형식의 분석 |
| `chain-of-thought` | Chain of Thought | 4 | 정책 효과 추론, 인과관계 분석 |

### 4.2 단원별 콘텐츠 구조
```json
{
  "slug": "zero-shot",
  "title": "Zero-shot Prompting",
  "description": "예시 없이 지시만으로 AI에게 정책 분석 작업을 요청하는 방법",
  "duration": "15분",
  "order": 2,
  "content": "## Zero-shot Prompting이란?\n...",
  "examples": [
    {
      "title": "정책 이해관계자 분석",
      "prompt": "한국의 '국민연금 개혁'에 대한 주요 이해관계자를 분석해주세요.\n\n다음 형식으로 작성:\n| 이해관계자 | 입장 | 주요 관심사 | 영향력 |\n\n최소 5개 이해관계자 그룹을 포함해주세요.",

    "starterPrompt": "기후변화 정책 설명해줘",

      "어떤 국가 또는 지역의 정책인가요?",

      "누구를 위한 분석인가요? (정책입안자, 연구자, 일반시민)"
    ]
  }
}
```
